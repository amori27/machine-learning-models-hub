# Contributing to Machine Learning Models Hub

Thank you for considering contributing to **Machine Learning Models Hub**! We welcome contributions of all kinds, including bug fixes, new features, documentation improvements, and more.

## Getting Started

1. Fork the repository.
2. Clone your fork:
   ```bash
   git clone https://github.com/your-username/machine-learning-models-hub.git
   ```
3. Create a feature branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
5. Make your changes.
6. Run tests:
   ```bash
   python -m pytest tests/
   ```
7. Commit and push, then open a pull request.

## Code Style

- Follow [PEP 8](https://peps.python.org/pep-0008/).
- Use descriptive variable and function names.
- Write docstrings for all public functions and classes.

## Pull Request Process

1. Ensure all tests pass.
2. Update `README.md` if your change affects the public interface.
3. Update `CHANGELOG.md` with a brief description of your change.
4. Your PR will be reviewed by a maintainer.

## Reporting Issues

Report bugs and request features via [GitHub Issues](https://github.com/amori27/machine-learning-models-hub/issues).

## Code of Conduct

Please note that this project adheres to a [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you agree to uphold its terms.
